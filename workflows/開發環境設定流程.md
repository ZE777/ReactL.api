# 開發環境設定流程

## 前置需求

- .NET 8 SDK
- SQL Server（LocalDB 或完整版）
- Visual Studio 2022 或 VS Code + C# 擴充

## 首次設定

```bash
# 1. 還原套件
dotnet restore

# 2. 設定連線字串（本機開發）
# 編輯 appsettings.Development.json
{
  "ConnectionStrings": {
    "DefaultConnection": "Server=(localdb)\\mssqllocaldb;Database=PromptStudioDb;Trusted_Connection=True;"
  }
}

# 3. 建立資料庫
dotnet ef database update --project ReactL.api

# 4. 啟動專案
dotnet run --project ReactL.api
```

## Swagger 測試介面

開發模式下可在以下網址測試 API：
```
https://localhost:{port}/swagger
```

## 日誌資料夾（logs/）

首次執行後會自動在專案根目錄建立 `logs/` 資料夾，  
Serilog 每日自動產生 `log-{yyyyMMdd}.txt`。

## 環境設定檔說明

| 檔案 | 用途 |
|------|------|
| `appsettings.json` | 共用設定（Log Level、AI Model 預設值） |
| `appsettings.Development.json` | 本機開發設定（DB 連線、API Key） |

API Key 等敏感資訊使用 `.NET User Secrets` 管理，不寫入檔案：

```bash
dotnet user-secrets set "AiSettings:ApiKey" "your-key-here"
```
