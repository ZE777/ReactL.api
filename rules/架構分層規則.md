# 架構分層規則

## 資料夾結構

```
ReactL.api/
├── Controllers/     ← HTTP 進入點，只做請求接收與回應
├── Interfaces/      ← 所有 IService、IRepository 介面
├── Services/        ← 商業邏輯實作
├── Repositories/    ← 資料存取實作（EF 查詢）
├── Models/          ← EF Entity（對應資料表）
├── DTOs/            ← Request / Response 物件
├── Data/            ← DbContext
├── Migrations/      ← EF 自動產生，不手動修改
├── SqlScripts/
│   ├── Init/        ← 初始化腳本
│   └── Changes/     ← 結構異動腳本
├── Middleware/      ← 錯誤處理、JWT、Request Logging
├── Common/          ← ApiResponse、常數、Helper
└── logs/            ← Serilog 產生，不進 Git
```

## 層與層的依賴方向

```
Controller → IService (介面)
Service    → IRepository (介面)
Repository → DbContext (EF)
```

- Controller 只能依賴 `Interfaces/` 裡的 `IXxxService`，不直接呼叫 Repository
- Service 只能依賴 `Interfaces/` 裡的 `IXxxRepository`，不直接使用 DbContext
- Repository 是唯一可以直接使用 `DbContext` 的地方

## 各層職責邊界

| 層 | 可以做 | 不可以做 |
|----|--------|----------|
| Controller | 接收請求、驗證 ModelState、呼叫 Service、回傳 ApiResponse | 寫商業邏輯、直接查資料庫 |
| Service | 商業邏輯、呼叫 Repository、呼叫外部 API（AI）| 直接操作 DbContext、處理 HTTP |
| Repository | EF CRUD 查詢 | 寫商業邏輯、呼叫其他 Service |
| Middleware | 攔截請求/回應、統一錯誤格式 | 寫商業邏輯 |
