# 命名規範

## 檔案命名

| 類型 | 規則 | 範例 |
|------|------|------|
| Controller | `{Entity}Controller` | `PersonaController` |
| Service 介面 | `I{Entity}Service` | `IPersonaService` |
| Service 實作 | `{Entity}Service` | `PersonaService` |
| Repository 介面 | `I{Entity}Repository` | `IPersonaRepository` |
| Repository 實作 | `{Entity}Repository` | `PersonaRepository` |
| EF Entity | 單數名詞 | `Persona`, `Conversation`, `BotBinding` |
| DTO Request | `{Action}{Entity}Request` | `CreatePersonaRequest`, `UpdateBotRequest` |
| DTO Response | `{Entity}Response` | `PersonaResponse`, `ConversationResponse` |
| DbContext | `AppDbContext` | — |

## C# 命名規則

- **Class、Property、Method**：PascalCase
- **private field**：`_camelCase`（底線前綴）
- **參數、區域變數**：camelCase
- **介面**：`I` 前綴，例如 `IPersonaService`
- **常數**：PascalCase，例如 `DefaultModel`

## SQL Scripts 命名

```
Init/
  V001_CreateSchema.sql
  V002_SeedRoles.sql

Changes/
  V001_AddBotTokenColumn.sql
  V002_AlterConversationAddPersonaId.sql
```

格式：`V{版本號}_{動詞}{目標}.sql`  
版本號 3 碼補零，Init 與 Changes 各自獨立編號。

## API 路由規範

- 全部小寫 kebab-case
- 使用複數資源名稱
- 版本前綴：`/api/v1/`

```
GET    /api/v1/personas
GET    /api/v1/personas/{id}
POST   /api/v1/personas
PATCH  /api/v1/personas/{id}
DELETE /api/v1/personas/{id}
```
