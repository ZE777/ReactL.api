# 日誌規則（Serilog）

## 設定

- 套件：`Serilog.AspNetCore`、`Serilog.Sinks.File`、`Serilog.Sinks.Console`
- 每日一檔：`logs/log-{Date}.txt`，同一天持續 append，不覆蓋
- `logs/` 資料夾加入 `.gitignore`，不進版控

## 必須記錄的時機

| 時機 | Log Level | 內容 |
|------|-----------|------|
| 每支 API 請求進入 | Information | Method、Path、QueryString |
| 每支 API 回應完成 | Information | StatusCode、耗時(ms) |
| Service 層執行關鍵動作 | Information | 動作描述、關聯 ID |
| 呼叫外部 AI API | Information | Model、Token 用量 |
| 驗證失敗 / 業務邏輯錯誤 | Warning | 錯誤原因、相關參數 |
| 未預期例外 | Error | 完整 Exception 堆疊 |

## 使用方式

```csharp
// 注入
public class PersonaService : IPersonaService
{
    private readonly ILogger<PersonaService> _logger;

    public PersonaService(ILogger<PersonaService> logger)
    {
        _logger = logger;
    }

    public async Task<PersonaResponse> GetByIdAsync(int id)
    {
        _logger.LogInformation("取得 Persona，ID: {PersonaId}", id);
        // ...
    }
}
```

## 結構化日誌參數

- 使用具名佔位符：`{PersonaId}` 而非字串串接
- 敏感資料（Token、密碼）不得寫入 Log

## 請求日誌中介層

由 `Middleware/RequestLoggingMiddleware.cs` 統一處理所有進出 log，  
各 Controller 不需要再手動記錄 HTTP 層的進出流量。
